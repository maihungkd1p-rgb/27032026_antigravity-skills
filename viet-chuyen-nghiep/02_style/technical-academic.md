# Technical Academic - Phương Pháp Trình Bày

**Module:** Output  
**Mục đích:** Tạo tài liệu kỹ thuật, học thuật, chuyên môn với cấu trúc chặt chẽ và giọng văn uy tín

## Khi Nào Sử Dụng

**Load method này khi:**
- ✅ Loại content: Tài liệu kỹ thuật, sách giáo khoa, whitepaper, documentation
- ✅ Mục đích: Giáo dục, giải thích hệ thống phức tạp, tài liệu tham khảo
- ✅ Độc giả: Professionals, kỹ sư, học viên, nghiên cứu viên
- ✅ Platform: Technical documentation, academic publications, training materials
- ✅ Goal: Chuyển tải kiến thức chính xác, có cấu trúc, dễ tra cứu

**Không dùng khi:**
- Cần emotional connection (dùng storytelling)
- Quick how-to guides đơn giản (dùng technical)
- Executive brief ngắn gọn (dùng executive)
- Blog/social media casual (dùng storytelling)

---

## Nguyên Tắc Cốt Lõi

**Technical Academic = Clarity (Rõ ràng) × Structure (Cấu trúc) × Authority (Uy tín)**

Đặc điểm then chốt:
- **Clarity:** Mỗi câu, mỗi đoạn có mục đích rõ ràng
- **Structure:** Logic chặt chẽ, dễ scan và tra cứu
- **Authority:** Giọng văn khách quan, chuyên môn, đáng tin cậy

## Phần 1: Quy Tắc Cấu Trúc Đoạn Văn

### Nguyên Tắc Biến thiên Có Chủ đích (Intentional Variation)

**Mỗi đoạn văn: Từ 2-5 câu, biến thiên theo mục đích**

Đây là hướng dẫn linh hoạt, ưu tiên sự tự nhiên hơn quy tắc cứng. Điều quan trọng là các đoạn KHÔNG nên đều nhau về độ dài.

**Nguyên tắc phân phối:**
- Tránh pattern lặp (vd: 3-3-3-3 câu liên tiếp)
- Xen kẽ đoạn ngắn và đoạn dài
- Đoạn ngắn tạo điểm dừng, đoạn dài mang tải nội dung chính

**Quy tắc xử lý:**
```
Đoạn 1-2 câu: ✅ Transition, emphasis, hoặc điểm nhấn quan trọng
Đoạn 3-4 câu: ✅ Giải thích chính, phát triển ý tưởng
Đoạn 5-6 câu: ✅ Phân tích sâu, giữ mạch logic liên tục
Đoạn 7+ câu: ⚠️ Cân nhắc tách, trừ khi ý tưởng không thể ngắt
```

**Ví dụ phân phối tốt trong một section:**
```
[Đoạn 2 câu - Mở đề bằng tuyên bố]
[Đoạn 5 câu - Giải thích chi tiết cơ chế]
[Đoạn 3 câu - Ví dụ minh họa]
[Đoạn 1 câu - Câu chuyển tiếp]
[Đoạn 4 câu - Kết luận và mở rộng]
```

**Ví dụ từ Chapter 2:**

✅ **Chuẩn (2 câu):**
```
Ngày nay, chúng ta chứng kiến sự ra đời của những mô hình ngôn ngữ lớn (LLM) 
với khả năng vượt trội trong suy luận và giải quyết vấn đề. Trên lý thuyết, 
chúng đã sẵn sàng để tiếp quản nhiều công việc phức tạp.
```

✅ **Chuẩn (3 câu - giải thích):**
```
Workflow giống như Quy trình Vận hành Tiêu chuẩn (SOP). Nó là một danh sách 
các bước tuần tự phải thực hiện để hoàn thành một nhiệm vụ.

Điểm khác biệt quan trọng: Workflow là quy trình thủ công. Nó không tự chạy. 
Người dùng phải chủ động kích hoạt bằng một lệnh cụ thể.
```

❌ **Sai (đoạn quá dài - cần tách):**
```
AI Agent có khả năng mạnh mẽ trong việc xử lý dữ liệu. Nó có thể đọc nhiều 
file cùng lúc. Nó có thể phân tích và tổng hợp thông tin. Nó có thể tạo 
báo cáo tự động. Nó có thể lưu file vào đúng vị trí. Nó có thể chia sẻ 
kết quả với team. Tất cả điều này giúp tiết kiệm thời gian.
```

**Cách sửa:** Tách thành 2-3 đoạn theo chủ đề logic.

---

### Topic Sentence (Câu Chủ Đề)

**Câu đầu tiên của mỗi đoạn = Topic Sentence**

Nguyên tắc:
- Nêu rõ ý chính của đoạn
- Các câu sau: phát triển, minh chứng, hoặc giải thích ý đó
- Đừng chôn ý chính ở giữa hay cuối đoạn

**Pattern chuẩn:**
```
[Topic Sentence - Ý chính]
[Supporting Sentence 1 - Phát triển]
[Supporting Sentence 2 - Minh chứng/Giải thích]
[Optional: Closing Sentence - Kết luận đoạn]
```

**Ví dụ:**

✅ **Topic sentence rõ ràng:**
```
Antigravity giải quyết bằng hệ thống Artifacts — bằng chứng công việc mà 
Agent tạo ra trong quá trình thực hiện nhiệm vụ.

Trước khi làm việc phức tạp, Agent trình bày kế hoạch — checklist các bước 
dự định thực hiện — để bạn duyệt. Trong quá trình làm việc, Agent chụp ảnh 
màn hình các bước quan trọng.
```

❌ **Không có topic sentence:**
```
Agent chụp ảnh màn hình. Nó cũng ghi video. Log được lưu chi tiết. Đây là 
một hệ thống artifacts.
```

## Phần 2: Dòng Chảy Logic

### Kết Nối Giữa Các Đoạn

Mỗi đoạn không đứng độc lập. Chúng phải kết nối tự nhiên qua:

**1. Từ nối (Transition Words)**
```
- Tuy nhiên, Nhưng → Contrast
- Do đó, Vì vậy → Consequence
- Hơn nữa, Ngoài ra → Addition
- Ngược lại → Opposition
- Tương tự → Similarity
```

**2. Tham chiếu ngữ cảnh**
```
- "Điều này..."
- "Vấn đề trên..."
- "Cơ chế này..."
- "Khả năng đó..."
```

**3. Logic progression**
```
- Problem → Solution
- General → Specific
- Theory → Practice
- Question → Answer
```

**Ví dụ kết nối tốt:**

```
Chatbot 2022 chỉ giải quyết được phần xử lý — phần giữa của vòng đời.

AI Agent 2025 mở rộng thêm phần đóng gói — có thể tạo file thay vì chỉ văn 
bản trong chat.

Nhưng thu thập, lưu trữ, phân phối, tái sử dụng — bốn giai đoạn quan trọng 
khác — tất cả vẫn là việc của con người.
```

**Phân tích:** 
- Đoạn 1: Nêu giới hạn chatbot
- Đoạn 2: Transition "AI Agent 2025" → tiến bộ
- Đoạn 3: Transition "Nhưng" → vẫn còn gap

## Phần 3: Hạn Chế Bullet Points

### Nguyên tắc: Văn xuôi > Bullet points

**Bullet points CHỈ dùng cho:**
- Checklist kỹ thuật
- Liệt kê tham số/spec
- Danh sách định nghĩa
- Code examples/commands

**Thân văn giải thích: BẮT BUỘC văn xuôi liền mạch**

❌ **Tránh pattern này trong giải thích:**
```
Knowledge có những đặc điểm:
- Tự động học từ hội thoại
- Lưu trữ dài hạn
- Giúp Agent nhớ kinh nghiệm
```

✅ **Viết văn xuôi:**
```
Knowledge trong Antigravity tương đương với kinh nghiệm làm việc thực tế. 
Đây là loại tri thức được tích lũy dần qua quá trình tương tác, giống như 
cách nhân viên mới "học lỏm" văn hóa và thói quen của sếp.
```

**Khi nào được dùng bullet:**

✅ **Technical specs:**
```
Yêu cầu hệ thống:
- RAM: 8GB trở lên
- Storage: 20GB khả dụng
- OS: Windows 10/11, macOS 12+
```

✅ **Checklist:**
```
Trước khi deploy:
- [ ] Chạy test suite
- [ ] Review security rules
- [ ] Backup database
```

## Phần 4: Cấu Trúc Heading

### Phân Cấp Tiêu Đề

**H1 (Tiêu đề chính) - Cho Sách Giáo Khoa:**
```markdown
# 3.1 Tên bài học (Viết hoa chữ đầu)
```
*Lưu ý: Format này áp dụng cho tài liệu dạng sách/giáo trình với hệ thống đánh số chương.*

**H1 (Tiêu đề chính) - Cho Blog/Article độc lập:**
```markdown
# IN HOA TOÀN BỘ (tùy chọn)
# Hoặc Viết Hoa Chữ Đầu
```

**H2, H3, H4... (Tiêu đề phụ):**
```markdown
## Chỉ viết hoa chữ đầu tiên
### Tương tự với H3
```

### Format Tiêu Đề: Noun Phrase

**Tránh:**
```
❌ 2.4.2 Knowledge: Khi nào Agent tự học?
❌ Làm thế nào để thiết kế Workflow?
```

**Dùng:**
```
✅ 2.4.2 Tầng Knowledge (Giai đoạn Khám phá)
✅ Phương pháp thiết kế Workflow
✅ Cơ chế tự động học
```

**Nguyên tắc:** Tiêu đề là cụm danh từ ngắn gọn, không phải câu hỏi hay câu hoàn chỉnh.

## Phần 5: Giọng Văn & Tone

### Đặc Điểm Giọng Văn

**Technical Academic tone:**
- Khách quan, không cảm xúc thái quá
- Chuyên môn nhưng không hàn lâm khô khan
- Uy tín nhưng không xa cách
- Rõ ràng nhưng không đơn giản hóa quá mức

**Ví dụ so sánh:**

❌ **Quá casual (như blog):**
```
Thật tuyệt vời! AI giờ đây có thể tự động làm mọi thứ cho bạn. Bạn sẽ không 
tin được những gì nó làm được đâu!
```

❌ **Quá khô khan (academic cứng nhắc):**
```
Nghiên cứu cho thấy rằng việc ứng dụng trí tuệ nhân tạo vào quy trình vận 
hành doanh nghiệp mang lại hiệu quả tối ưu hóa đáng kể theo các chỉ số đo 
lường năng suất.
```

✅ **Chuẩn Technical Academic:**
```
AI Agent có khả năng tự động hóa nhiều khâu trong quy trình xử lý dữ liệu. 
Từ thu thập thông tin, phân tích, đến đóng gói kết quả — tất cả có thể 
diễn ra mà không cần can thiệp thủ công.
```

### Tránh AI-style

Đã được quy định trong `quy-tac-viet.md`, nhắc lại:

❌ **Tránh:**
```
- "Here are 5 key points..."
- "It's important to note that..."
- "In conclusion..."
- "Let me explain..."
```

✅ **Dùng:**
```
- Viết trực tiếp: "Có năm điểm chính..."
- "Cần lưu ý rằng..."
- "Tóm lại..." / Kết thúc tự nhiên
- Giải thích trực tiếp, không cần "tín hiệu"
```

## Phần 6: Workflow Thực Hiện

### Bước 1: Lập Outline Logic (15 phút)

**Xây dựng cấu trúc:**
```
H1: [Chủ đề chính]
  H2: [Khái niệm nền tảng]
    H3: [Định nghĩa]
    H3: [Đặc điểm]
  H2: [Phân tích chi tiết]
    H3: [Thành phần 1]
    H3: [Thành phần 2]
  H2: [Ứng dụng thực tế]
```

**Nguyên tắc:**
- General → Specific
- Theory → Practice
- Problem → Solution

### Bước 2: Viết Topic Sentences (10 phút)

Cho mỗi section (H2, H3), viết topic sentence trước:
```
## [Heading]
[Topic sentence nêu ý chính của section này]
```

Làm điều này giúp đảm bảo logic flow trước khi viết chi tiết.

### Bước 3: Phát Triển Đoạn Văn (30 phút)

**Cho mỗi topic sentence:**
1. Viết 1-3 câu supporting
2. Kiểm tra: đoạn có quá 4 câu không? Nếu có → tách
3. Thêm từ nối với đoạn trước

**Pattern:**
```
[Topic sentence]
[Supporting 1]
[Supporting 2]
[Optional: Closing]

[Transition word/phrase] [Topic sentence đoạn tiếp theo]
...
```

### Bước 4: Review & Tighten (15 phút)

**Checklist:**
- [ ] Mỗi đoạn ≤ 4 câu?
- [ ] Mỗi đoạn có topic sentence rõ ràng?
- [ ] Các đoạn kết nối logic qua từ nối?
- [ ] Heading đều là noun phrase?
- [ ] Không có bullet points trong thân văn?
- [ ] Giọng văn chuyên môn, khách quan?
- [ ] Tuân thủ quy-tac-viet.md?

## Tóm Tắt Công Thức

**Technical Academic = Natural Variation + Topic Sentence + Logic Flow + Professional Tone**

**Must-have:**
1. Đoạn văn 2-6 câu, độ dài BIẾN THIÊN không đều nhau
2. Topic sentence mở đầu mỗi đoạn
3. Từ nối/tham chiếu giữa đoạn
4. Heading noun phrase
5. Bullet points có mục đích (specs, steps, comparisons)
6. Giọng văn chuyên môn, khách quan
7. Không bịa đặt tên người/công ty/thời gian cụ thể

**Workflow:**
Outline → Topic sentences → Phát triển đoạn (độ dài biến thiên) → Review flow → Kiểm tra natural variation

**Dùng khi:** Technical documentation, sách giáo khoa, whitepapers, training materials, academic publications

**Master method này cho:** Professional content cần clarity + structure + authority
